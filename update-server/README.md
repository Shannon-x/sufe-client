# update-server

Static manifests served from the project's `gh-pages` branch (i.e.
`https://shannon-x.github.io/sufe-client/...`). Two consumers:

- **Tauri desktop updater** â€” polls `desktop/latest.json`. Shape per
  [Tauri 2 updater plugin docs](https://tauri.app/plugin/updater/).
- **Android in-app updater** â€” polls `mobile/android/latest.json`.
  Custom shape, documented below.

Files under `dist/` are committed-source templates that ship to
`gh-pages` on every successful release. The actual `latest.json`
manifest is **regenerated by CI** from release artifacts (see
`.github/workflows/release.yml` â†’ `publish-desktop-manifest` and
`.github/workflows/mobile.yml` â†’ `publish-mobile-manifest`); the
templates here are starter content / first-deploy bootstrap.

## Tauri desktop manifest

```json
{
  "version": "0.2.0",
  "notes": "Tag v0.2.0",
  "pub_date": "2026-09-27T12:00:00Z",
  "platforms": {
    "darwin-aarch64":   { "url": "...", "signature": "..." },
    "darwin-x86_64":    { "url": "...", "signature": "..." },
    "linux-x86_64":     { "url": "...", "signature": "..." },
    "windows-x86_64":   { "url": "...", "signature": "..." }
  }
}
```

`signature` is the ed25519 detached signature emitted by
`tauri-action`; the public key sits in `desktop/src-tauri/tauri.conf.json`
under `plugins.updater.pubkey`. CI imports the matching private key
from the repo secret `TAURI_SIGNING_PRIVATE_KEY`.

To wire the Tauri updater, set this URL in
`desktop/src-tauri/tauri.conf.json`:

```json
"plugins": {
  "updater": {
    "endpoints": [
      "https://shannon-x.github.io/sufe-client/desktop/latest.json",
      "https://github.com/Shannon-x/sufe-client/releases/latest/download/latest.json"
    ],
    "pubkey": "..."
  }
}
```

The second endpoint is a fallback in case gh-pages is briefly stale â€”
Tauri tries them in order and uses the first one that returns a
manifest with a higher `version`.

## Android manifest

```json
{
  "version": "0.2.0",
  "url": "https://github.com/.../xboard-v0.2.0-android.apk",
  "sha256": "deadbeef...",
  "min_version": "0.1.0",
  "force": false,
  "notes": "Tag v0.2.0"
}
```

The Android client polls this on app start (skipped on debug builds);
when `version > BuildConfig.VERSION_NAME`, it offers an in-app update.
The downloaded APK's sha256 is verified before install so the user
never sees a tampered binary even if the GitHub release is mirrored.
`force: true` blocks app launch until the update is installed (used
for security fixes only).
